Introduction

Welcome!

This book is intended to bridge the gap between what you learned in a Web development bootcamp and what it takes to build and operate backends for real-world products.

You already know how to build a backend for a Web application: you write some code, you put it in a server somewhere, you configure some database, and you hit 'run'. Simple enough, no?

But, when you join a medium-sized project, you'll find that, typically, things are not so simple. We set out to protect ourselves from all kinds of failure scenarios by ensuring our backends don't just work, but are robust.

For example, have you ever considered what happens to your backend...

  • ... when you have 10x, 100x, 1000x more requests to your server?
  • ... when the database disk catches fire?
  • ... when the datacenter hosting your server has an outage?

To address these kind of failures, and others more mundane but also more subtle, backends tend to get complicated: lots of middle layers, lots of different tools everywhere, architecture diagrams with lots of boxes and arrows, strange artifacts in the code, intricate procedures to do things that, for simple systems, would be a couple of commands away.

This complexity can be overwhelming at first. It's not obvious to understand what's the purpose of everything, how it all fits together, why things are done that particular way, and which are the implications. So that this transition is easier, and you can become a productive, knowledgeable contributor to the team as soon as possible, we'll learn all this from the ground up.

We won't teach you about specific technologies. Depending on the team you end up joining, you will find different langugages, databases, deployment systems, all kinds of different buzzwords that will sound alien to you. You can teach yourself those comparatively easily, if needed.

Rather, we'll focus on the commonalities: the building principles for which all those technologies are nothing but implementations. Tools come and go, but principles remain. Plus, tools cannot shield you from those principles: some are fundamental in nature, and permeate every single component of a production-grade Web architecture.

We'll go through the theory, and then apply it to a real project.

Requirements

Basic knowledge of JavaScript, Node.js, Express.js, MongoDB, and the command line interface is assumed.
